## 3. Team Structure вҖ” Solo Developer Version

When building as a solo developer, you wear all hats. Here is how to organize your time:

### 3.1 Weekly Role Rotation

| Week Type | Primary Focus | Secondary |
|-----------|--------------|-----------|
| Implementation Week | 70% coding | 20% testing, 10% docs |
| Integration Week | 50% coding | 30% testing, 20% integration |
| Review Week | 30% coding | 40% testing, 30% documentation |

### 3.2 Daily Schedule (Solo Dev)

```
09:00вҖ“09:30  Review yesterday, plan today's tasks
09:30вҖ“12:00  Deep work: core implementation

15:00вҖ“16:00  Testing / debugging
16:00вҖ“16:45  Documentation / comments

в”ңв”Җв”Җ hotfix/phase-3-texture-crash
в””в”Җв”Җ release/v0.1.0
```

### 4.2 Commit Convention

All commits follow **Conventional Commits**:

```
feat(ecs): add archetype migration for component add/remove
fix(renderer): resolve texture binding crash on resize
perf(physics): optimize broadphase BVH insertion to O(log n)
docs(scripting): add Lua API reference for Vec3
test(audio): add unit tests for spatial attenuation
refactor(memory): extract pool allocator to separate file
chore(build): update CMake minimum version to 3.28
```
